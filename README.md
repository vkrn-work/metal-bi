# EMK BI — дашборд «Конверсия КП → Заказ»

Веб-приложение поверх конвейера `emk_bi`: вы загружаете выгрузки КП и ЗК из CRM,
они превращаются в единую базу, база показывается интерактивным отчётом.
Вход по общему паролю.

## Что умеет

- **Вход по паролю** — один общий пароль в переменной `APP_PASSWORD`, сессия в подписанной cookie на 12 часов.
- **Отчёт** — KPI конверсии по количеству и по деньгам со сравнением с предыдущим периодом
  той же длины, таблица категорий металлопродукции с раскрытием до 5 уровня,
  два кольца долей (по КП и по Заказам), динамика конверсий по полугодиям.
- **Фильтры** — период, источник заявки и пять атрибутов клиента + выборка.
- **Обновление данных** — страница `/upload`: полный пересчёт из xlsx или быстрая замена готового `base.parquet`.

## Правила расчёта

| Показатель | Как считается |
|---|---|
| COUNTUNIQUE | уникальные `doc_number` внутри группы |
| SUM из Маржа | сумма `margin_eur` (всегда EUR) |
| CR КП-ЗК | уникальные ЗК / уникальные КП |
| CR деньги | маржа ЗК / маржа КП |
| Доли категорий | доля категории в сумме по всем категориям 1 уровня |

Итог в таблице считается по всей выборке, а не суммой строк: один документ может
попадать сразу в несколько категорий, поэтому сумма по строкам больше итога.
Это то же поведение, что в исходном отчёте.

## Структура

```
app/
  main.py          FastAPI: вход, страницы, API, загрузка файлов
  report.py        агрегация базы через DuckDB
  pipeline.py      фоновый запуск конвейера над загруженными файлами
  templates/       login.html, dashboard.html, upload.html
  static/          styles.css, app.js, upload.js
etl/               конвейер подготовки базы (prepare_base.py, config.py, classify.py, references/)
data/              base.parquet — рабочая база (в git не хранится)
```

## Локальный запуск

```bash
./run_local.sh          # http://127.0.0.1:8000, пароль по умолчанию emk2026
```

или вручную:

```bash
pip install -r requirements.txt
APP_PASSWORD=ваш_пароль uvicorn main:app --app-dir app --port 8000
```

Положите готовый `base.parquet` в `data/` — или загрузите его через `/upload` после старта.

## Переменные окружения

| Переменная | Назначение | По умолчанию |
|---|---|---|
| `APP_PASSWORD` | пароль входа | `emk2026` |
| `SECRET_KEY` | ключ подписи cookie (без него сессии слетают при рестарте) | случайный |
| `DATA_DIR` | где лежит рабочая база | `./data` |
| `PORT` | порт | задаёт Railway |

## Настройка фильтров

Соответствие «Клиент 1..5» и «Выборка» полям базы задаётся словарём `CLIENT_FILTERS`
в `app/report.py` — поменяйте пары там, если нужны другие атрибуты клиента.

Деплой на Railway — в [DEPLOY.md](DEPLOY.md).
