# Деплой на Railway

Займёт минут 10. Карта не нужна на пробном плане, но том для хранения базы
доступен только на платном (Hobby, ~5 $/мес) — без тома загруженная база
пропадёт при следующем передеплое.

## Шаг 1. Дозалить конвейер классификации

В репозиторий не попали два объекта — их нужно добавить один раз вручную
(файл `classify.py` слишком большой, чтобы залить его через интеграцию):

```bash
git clone https://github.com/vkrn-work/metal-bi.git
cd metal-bi

# из вашей папки emk_bi:
cp /путь/к/emk_bi/classify.py        etl/
cp /путь/к/emk_bi/references/*.csv   etl/references/

git add etl && git commit -m "ETL: классификатор и справочники" && git push
```

Без этого шага приложение работает, но кнопка «Пересобрать базу» из xlsx будет
заблокирована — останется только загрузка готового `base.parquet`.

## Шаг 2. Создать проект в Railway

1. [railway.app](https://railway.app) → войти через GitHub.
2. **New Project → Deploy from GitHub repo → `vkrn-work/metal-bi`**.
3. Railway сам определит Python и поднимет приложение (`railway.json` уже в репозитории).

## Шаг 3. Переменные окружения

**Variables** → добавить:

| Переменная | Значение |
|---|---|
| `APP_PASSWORD` | ваш пароль для входа в отчёт |
| `SECRET_KEY` | любая длинная случайная строка |
| `DATA_DIR` | `/app/data` |

`PORT` Railway подставляет сам — задавать не нужно.

## Шаг 4. Том для базы (рекомендуется)

**Settings → Volumes → New Volume**, mount path `/app/data`.

Без тома база живёт до следующего передеплоя. С томом — загруженные данные сохраняются.
Размера 1 ГБ хватает с большим запасом: полугодовая база занимает около 2 МБ.

## Шаг 5. Домен

**Settings → Networking → Generate Domain**. Получите адрес вида
`metal-bi-production.up.railway.app` — по нему открывается страница входа.

## Шаг 6. Залить данные

Откройте `/upload` на своём домене:

- **Быстрый путь** — приложить готовый `base.parquet`, который вы уже собрали локально.
  Отчёт оживёт через пару секунд.
- **Полный пересчёт** — приложить выгрузки КП и ЗК (можно несколько файлов сразу)
  и, если менялись, справочники клиентов и менеджеров. Конвейер идёт в фоне,
  лог виден прямо на странице. Полугодовая выгрузка КП на 111 тыс. строк
  считается несколько минут — страницу можно не держать открытой.

База подменяется только после успешной сборки; предыдущая версия сохраняется рядом как `.bak`.

## Обновление в следующем периоде

Просто зайдите на `/upload` и приложите новые файлы. Передеплой не нужен.

## Если что-то пошло не так

| Симптом | Причина |
|---|---|
| «Полный пересчёт недоступен» | не выполнен шаг 1 — нет `etl/classify.py` или пустая `etl/references/` |
| База пропала после передеплоя | не подключён том из шага 4 |
| Разлогинивает при каждом рестарте | не задан `SECRET_KEY` |
| Пересборка падает на чтении xlsx | в выгрузке переименованы столбцы — поправьте `etl/config.py` |
| Не хватает памяти при пересборке | поднимите план Railway: чтение xlsx на 111 тыс. строк требует около 1 ГБ |
