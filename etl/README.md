# ETL — конвейер подготовки базы

Здесь лежит тот же конвейер, что вы гоняете локально. Приложение вызывает
`prepare_base.py` в фоне, когда вы загружаете выгрузки на странице `/upload`.

## Что должно быть в этой папке

| Файл | В репозитории | Назначение |
|---|---|---|
| `prepare_base.py` | да | главный конвейер |
| `config.py` | да | соответствия столбцов выгрузок единой схеме |
| `classify.py` | **нет, добавьте вручную** | классификатор номенклатуры (Категории 1–5, тип металла) |
| `references/*.csv` | **нет, добавьте вручную** | справочники классификатора |

`classify.py` (около 65 КБ) и CSV-справочники не удалось залить через GitHub-интеграцию —
скопируйте их из вашей папки `emk_bi` и закоммитьте, см. шаг 1 в `DEPLOY.md`.

Пока их нет, приложение блокирует полный пересчёт и предлагает загрузить
готовый `base.parquet`. Проверка живёт в `app/pipeline.py::etl_available()`.

## Запуск вручную

```bash
python3 prepare_base.py --kp data/kp --zk data/zk \
  --clients data/clients.xlsx --managers data/managers.xlsx \
  --out ../data/base.parquet
```
